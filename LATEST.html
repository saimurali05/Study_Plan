INTERVIEW PLAYBOOK
Crack any technical interview
Applicable from Month 6 onwards — start applying by Month 7
Indian product companies (Razorpay, CRED, Zepto, Freshworks, Atlassian India) follow a predictable interview format. This playbook maps exactly what they test, what to say, and how to stand out from 1,000 other candidates who all know the same LeetCode.
// ROUND-BY-ROUND BREAKDOWN
📝
Round 1 — Online Assessment
2 DSA problems · 60–90 minutes · HackerRank/HireVue
WHAT THEY TEST
Arrays, strings, hashmaps, sliding window, two pointers. Rarely harder than LeetCode medium. Time complexity matters.
WINNING STRATEGY
Write brute force first (shows you understand the problem), then optimize. Partial credit exists. A commented brute force beats a broken optimal solution.
Tip: Read all test cases before writing a single line. Many fail by rushing.
PREP PLAN
Neetcode 150 list in order. 45 mins/day from Day 1. Focus on patterns: sliding window, two pointers, BFS/DFS, dynamic programming basics.
🧑‍💻
Round 2 — Technical Deep Dive
DSA + System Design basics · 60 mins · Zoom/Meet
DSA LIVE CODING
Think out loud before coding. Say "I'm thinking about a hashmap approach — that gives O(n) time." Interviewers want to see your thinking, not just the answer.
SYSTEM DESIGN (SDE-1)
URL shortener, rate limiter, chat system basics. Know: REST vs WebSocket, SQL vs NoSQL tradeoffs, caching with Redis, load balancer basics.
You don't need perfect answers — you need structured thinking. Use: Requirements → Scale → Data model → APIs → Components.
AI/ML QUESTIONS (2026 trend)
Know: what is RAG, how LLMs work at a high level, what embeddings are, when to use fine-tuning vs prompting. These are differentiators now.
🏗️
Round 3 — Project / Take-home
Build a mini-feature · 24–72 hrs · Code review follows
WHAT WINS THIS ROUND
Clean code structure, meaningful commit messages, proper error handling, a README that explains your decisions. Code quality over feature count every time.
TYPICAL ASKS
Build a REST API with auth. Add a feature to a codebase. Create a React component with tests. Build a simple AI-powered endpoint.
Always ask: "Should I prioritize features or code quality?" — shows maturity.
YOUR EDGE (by Month 6)
You've shipped 10+ real projects. You have patterns for FastAPI + Supabase, auth flows, AI integrations. You build faster and cleaner than candidates who only did tutorials.
🤝
Round 4 — Hiring Manager + Culture
Behavioral + motivation · 30–45 mins · Make or break
THE STAR METHOD
Situation → Task → Action → Result. Every behavioral answer needs a number: "reduced load time by 40%", "acquired 3 clients in 2 weeks", "shipped in 14 days instead of 30".
QUESTIONS THEY ALWAYS ASK
"Tell me about a project you're most proud of." / "How do you handle ambiguous requirements?" / "Why this company specifically?" / "What's your biggest technical failure and what did you learn?"
YOUR POWER MOVE
End with: "I noticed you're building [X feature/product]. I actually built something similar — here's the link." Demonstrates initiative, research, and relevance in one move.
Prep a company-specific answer. Laziness here costs offers.
